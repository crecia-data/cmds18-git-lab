# 05. 무료 배포(Pages) — 내 사이트를 세상에 공개

> **한 줄 목표**: GitHub Pages로 내 프로필 페이지를 **무료로 웹에 공개**하고, 진짜 링크를 손에 쥔다.
> **소요**: 약 10분 · **앞 단계**: `/step 4` · **이번에 쓰는 것**: `/site`

---

## 🤔 왜 신날까?

지금까지 만든 페이지는 내 컴퓨터에만 있었습니다. 이번 단계가 끝나면 **`https://내아이디.github.io/...`** 라는 **진짜 인터넷 주소**가 생기고, 누구에게나 링크로 보낼 수 있습니다. 그것도 **공짜로**.

## 💡 개념: Pages = 무료 홈페이지 발행 버튼

> - **GitHub Pages** = 내 저장소의 `index.html`을 그대로 **웹사이트로 발행**해주는 무료 기능.
> - 별도 서버·호스팅 비용 없이, 저장소만 있으면 됩니다.

---

## 🛠️ 따라하기

### ① 띄우기 — `/site`
```
사람들한테 보여주게 사이트로 띄워줘
```
**화면에서 일어나는 일**: Claude가 ① **공개 조건**을 안내하고(아래 참고) ② `index.html`을 Pages가 찾을 위치로 맞춘 뒤 ③ Pages를 켭니다.

### ② 공개로 전환 (무료 조건)
GitHub Pages는 **공개(public) 저장소면 무료**입니다. 지금 저장소가 비공개(private)면 → **공개로 바꿔도 되는지** Claude가 물어봅니다. (프로필 페이지라 보통 공개 OK — 단, 개인정보가 없는지 한 번 확인)

### ③ 접속
1~2분 뒤 알려준 주소(`https://내아이디.github.io/저장소/`)로 접속. **휴대폰으로도** 열어보세요. *"내 페이지가 진짜 인터넷에 떴다!"* 🎉

---

## ⚠️ 흔한 실수 / FAQ
- **"Pages가 안 켜져요"** → 저장소가 private이면 무료로는 안 됩니다. **public 전환**(공개해도 되는 내용인지 먼저 확인) 또는 유료 플랜.
- **빈 화면이 떠요** → 반영에 1~2분 걸립니다. 그래도 비면 `index.html` 위치·파일명을 확인.
- **링크가 404** → 주소의 저장소 이름·대소문자 확인.

## 🎤 강사 포인트
- **검증된 안심 멘트**: *"공개 저장소면 Pages·Actions 다 무료"* — 개인 무료 계정 Actions 월 2,000분, **public은 한도 미적용**(자료조사 검증값).
- 한도 주의: Pages 대역폭 100GB·빌드 10회/시는 **soft limit**, 상업/이커머스 용도는 약관상 금지 — 강의에서 "절대 상한"으로 단정하지 말 것.
- 결과물(라이브 URL)을 단톡방에 공유하게 하면 성취감·바이럴.

---

## ✅ 한 줄 정리
**공개(public) 저장소면 GitHub Pages로 무료 배포. "사이트로 띄워줘" 한마디면 링크가 생긴다.**
