# 01. 첫 저장소 만들기 — 내 프로필을 깃허브에 올리기

> **한 줄 목표**: `sample-project/`(내 프로필 페이지)를 내 GitHub 저장소로 만들어 올린다. **명령어는 Claude가 친다.**
> **소요**: 약 8분 · **앞 단계**: `gh` 로그인 완료(`github_setup/`) · **이번에 쓰는 것**: `/setup-check`, 자연어

---

## 🤔 왜 이걸 먼저?

내 컴퓨터 안에만 있는 파일은 — 노트북이 고장나면 끝, 친구에게 보여주기도 번거롭습니다. **GitHub**은 내 작업을 인터넷의 안전한 금고에 올려두고, 언제든 공유·복구·협업할 수 있게 해주는 곳입니다. AI로 뭔가 만들기 시작하면 **결국 여기에 올리게 됩니다.** 오늘 그 첫 단추를 끼웁니다.

## 💡 개념: 저장소 = 작업실 / 올리기 = 클라우드 금고

> - **repository(저장소, repo)** = 프로젝트를 **통째로 담는 작업실/서랍장**. 보통 1프로젝트 = 1저장소.
> - **push(푸시)** = 내 책상의 작업물을 **클라우드 금고에 사본으로 올리는 일**.

명령어(`git init`, `git push`…)는 외울 필요 없습니다. **"깃허브에 올려줘"** 한마디면 됩니다.

---

## 🛠️ 따라하기

### ① 준비 확인 — `/setup-check`
```
/setup-check
```
git·gh·로그인이 ✅인지 확인합니다. 로그인이 ❌면 *"깃허브 로그인 도와줘"* 또는 `github_setup/00_github_setup_guide.md`.

### ② 한 마디로 올리기
```
이 폴더를 깃허브 비공개 저장소로 만들어서 올려줘
```
**화면에서 일어나는 일**: Claude가 순서대로 ① `git init`(작업실 만들기) → ② 첫 저장(커밋) → ③ `gh repo create`(GitHub에 저장소 생성) → ④ `push`(올리기)를 실행하고, 만들어진 **저장소 주소(URL)**를 알려줍니다.

### ③ 눈으로 확인
알려준 URL을 브라우저로 열어보세요. **"내 파일이 인터넷에 올라가 있다!"** — 이 순간이 시작입니다.

---

## ⚠️ 흔한 실수 / FAQ
- **로그인 오류(Permission/auth)** → `gh auth login` 또는 *"깃허브 로그인 도와줘"*.
- **"이미 같은 이름 저장소가 있어요"** → 이름을 바꿔서 다시(예: `my-profile-2`).
- **비공개 vs 공개?** → 지금은 **비공개(private)**로 시작합니다. 나중에 배포(05단계)할 때 공개로 바꿉니다.

## 🎤 강사 포인트
- ③의 "내 게 인터넷에 올라갔다" 순간을 크게 짚어 동기 부여.
- `gh`가 인증·생성·푸시를 한 번에 대신해줌 → **"사람은 목적만, 도구가 절차를"**.
- 비공개로 시작하는 이유(보안 습관)를 한 줄 언급. 공개 전환은 05단계 예고.
