# 06. 규칙으로 굳히기 + 캡스톤 — 자연어를 '안전·매끄럽게'

> **한 줄 목표**: `CLAUDE.md`에 내 git 안전규칙을 박아, 매번 말하지 않아도 AI가 그 규칙대로 일하게 만든다. 그리고 전체 흐름을 한 바퀴 돌린다.
> **소요**: 약 12분 · **앞 단계**: `/step 5` · **이번에 쓰는 것**: 자연어, 전체 슬래시커맨드

---

## 🤔 왜 마지막에 이걸?

지금까지는 *"위험한 거 전엔 저장해줘"*를 **매번 말로** 시켰습니다. 그런데 이걸 **한 번 규칙으로 적어두면**, 다음부터는 AI가 알아서 그 규칙을 지킵니다. 매번 말하기 → **한 번 굳히기**. 이게 하네스(harness)의 핵심이고, 16회차에서 다룬 개념의 실생활 버전입니다.

## 💡 개념: CLAUDE.md = AI에게 주는 '집 규칙'

> - **`CLAUDE.md`** = 이 프로젝트(작업실)의 **벽에 붙은 집 규칙**. AI가 매 작업마다 이 규칙을 읽고 따른다.
> - 이 규칙은 **git으로 버전관리되는 컨텍스트** → 세션이 바뀌어도, 협업자가 와도 **같은 규칙** 적용.

---

## 🛠️ 따라하기

### ① 규칙 박기
```
앞으로 위험한 작업 전엔 항상 먼저 커밋하라고 규칙에 적어줘
main에는 직접 올리지 말고 꼭 PR로 하라고도 추가해줘
```
**화면에서 일어나는 일**: Claude가 `CLAUDE.md`를 열어 규칙을 추가하는 걸 보여줍니다. 이제 이 폴더에서는 **그 규칙이 기본값**이 됩니다.

### ② 슬래시커맨드 = 자연어 단축 버튼
지금까지 쓴 `/save`·`/undo`·`/experiment`·`/ship`·`/site`가 사실 **"자주 쓰는 자연어를 버튼으로 만든 것"**임을 확인합니다. 외울 필요 없이, 말로 해도 똑같이 동작합니다.

---

## 🎓 캡스톤 — 전체를 한 바퀴

배운 걸 한 문장으로 스스로 돌려보세요:
```
내 프로필 페이지를 고쳐서 → 실험 브랜치에서 디자인 바꾸고 → PR로 검토받아 반영하고 → 사이트로 배포까지 한 번에 진행해줘. 위험한 단계 전엔 저장하고, main엔 직접 올리지 말고 PR로.
```
→ **저장 → 실험 → 검토 → 배포**의 전체 흐름을, **명령어 하나 없이 말로** 완주합니다.
