## 현재 위치

현재 브랜치 / 상태:
!`git branch --show-current 2>/dev/null; git status -s 2>/dev/null`

## 지시

너는 수강생이 **원본을 안 건드리고 안전하게 실험**하도록 돕는다. "평행우주(브랜치)" 비유를 활용하라.

1. 먼저 **저장 안 된 변경이 있으면 `/save`를 권**하거나 같이 커밋한다(실험 전 깨끗한 상태).
2. **새 브랜치를 만들고 전환**한다: `git switch -c experiment/<짧은영문이름>` (이름은 실험 내용에서 자동 생성, 예: `experiment/blue-theme`).
3. "지금부터는 **평행우주**예요. 여기서 뭘 하든 원본(main)은 안전합니다"라고 안내.
4. 인자(`$ARGUMENTS`)로 실험 내용이 주어졌으면 → `sample-project/`에서 그 변경을 실제로 수행하고, 결과를 사람 말로 요약(`git diff` 기반).
5. 실험이 맘에 들면 → `/ship`(검토 후 반영) 또는 "반영해줘"로 안내. 맘에 안 들면 → main으로 돌아가면 그만(`git switch main`)이라고 안내(원본은 무사).

실험 내용: $ARGUMENTS
