## 되돌리기 기준점 확인

최근 저장점(커밋):
!`git log --oneline -5 2>/dev/null || echo "(아직 커밋이 없어 되돌릴 저장점이 없음)"`

지금 안 저장된 변경:
!`git status -s 2>/dev/null`

## 지시

너는 수강생을 안전하게 "마지막 저장점으로" 되돌려주는 역할이다. **되돌리기는 작업을 잃을 수 있으므로 반드시 확인 후 실행.**

1. 위 상태를 보고, **무엇을 어디로 되돌리는지 한 줄로 먼저 설명**한다(예: "지금 저장 안 된 변경을 버리고, 마지막 저장점 'OO'으로 돌아갈게요. 진행할까요?").
2. 커밋이 하나도 없으면 → 되돌릴 저장점이 없다고 안내하고, 대신 변경을 버릴지 물어본다.
3. 수강생이 동의하면:
   - 저장 안 된 변경만 버리고 마지막 커밋 상태로: `git restore .` (추적 파일) / 필요 시 새 파일 정리는 따로 확인.
   - 특정 과거 저장점으로 돌아가려면 어떤 커밋인지 먼저 확인받는다.
4. **`reset --hard`처럼 되돌릴 수 없는 명령은 쓰기 전에 한 번 더 확인**한다(이 키트에서는 차단돼 있을 수 있으니, 안전한 `git restore`/`git checkout`을 우선 사용).
5. 끝나면 "✅ 되돌리기 완료 — 마지막 저장점 상태입니다"라고 알려주고, "앞으로는 위험한 작업 전에 `/save`부터 하면 더 안전해요"라고 한 줄 덧붙여라.
