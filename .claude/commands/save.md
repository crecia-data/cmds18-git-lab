## 현재 변경 사항

!`git status -s 2>/dev/null || echo "(아직 git 저장소가 아님 — 먼저 /step 1)"`

변경 미리보기:
!`git diff --stat 2>/dev/null | tail -10`

## 지시

너는 수강생의 작업을 안전하게 "저장(커밋)"해 주는 역할이다.

1. 위 변경 내용을 보고, **무엇을 저장하는지 한 줄로 사람 말로 요약**해 알려준다(예: "프로필 제목과 색상 변경을 저장할게요").
2. 변경이 없으면 → "저장할 변경이 없어요"라고 알리고 끝낸다.
3. git 저장소가 아니면 → `/step 1`을 먼저 하라고 안내.
4. 저장 실행: `git add -A` 후 `git commit -m "<의미 있는 한국어 메시지>"`. **커밋 메시지는 변경 내용을 보고 의미 단위로 자동 작성**한다(예: "프로필 소개문구 수정", "배경색 파랑으로 변경").
5. 끝나면 "✅ 저장 완료 — 이제 마음껏 실험해도 돼요. 망치면 `/undo`로 여기로 돌아옵니다"라고 안심시켜라.

⚠️ `.env`·키 파일 등 민감정보가 변경에 섞여 있으면 커밋하지 말고 경고하라(.gitignore 확인).
