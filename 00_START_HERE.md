## 🆘 자주 막히는 점 FAQ

**Q. `/start`를 입력해도 반응이 없어요.**
→ 이 폴더(`cmds18-git-lab`) **안에서** `claude`를 실행했는지 확인하세요. 슬래시커맨드는 `.claude/commands/`가 있는 폴더에서만 보입니다.

**Q. `gh auth status`가 "not logged in"이라고 나와요.**
→ 아직 GitHub 로그인 전입니다. `gh auth login` 또는 Claude에게 "깃허브 로그인 도와줘"라고 하세요. (`github_setup/` 참고)

**Q. push할 때 권한 오류가 나요.**
→ 로그인 계정과 저장소 소유자가 맞는지 확인. `/setup-check`로 상태를 점검하세요.

**Q. 권한을 묻는 프롬프트가 자꾸 떠요.**
→ 실습 편의를 위해 `.claude/settings.json`에 안전한 git 명령(상태·diff·로그·커밋·브랜치)은 허용해 뒀습니다. 처음 보는 명령은 물어볼 수 있으니 안전하면 허용하세요. (`force push`·`rm -rf`는 막혀 있습니다.)

**Q. 실수로 다 망쳤어요.**
→ 걱정 마세요. 그게 오늘의 핵심입니다. Claude에게 "마지막 저장점으로 되돌려줘"(`/undo`)라고 하면 됩니다. **자주 저장(`/save`)이 최고의 안전망.**
